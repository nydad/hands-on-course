# WHY — North Star (Immutable)

## Purpose

`발표자료/presentation.html` (임원 대상 "챗봇에서 에이전트로" AI Hands-on 강의자료)을:

1. **선형적 내러티브**로 정리 — 일반 청중이 순서대로 따라가며 개념이 점진적으로 확장되도록
2. **시각적 가시성** 확보 — 125% 스케일, 은은한 포인트 팔레트
3. **과장 톤 제거** — 담백하고 절제된 표현
4. **내용 정확성 유지** — 사실 왜곡 없이 프레이밍만 재구성

## Success Criteria

- [ ] 내러티브: 챗봇 → 에이전트 → 멀티에이전트 → 분야 확산 → 역할 변화 → 현실 제약 (왕복 없음)
- [ ] 모든 슬라이드가 Chrome에서 검수되어 렌더링/링크/레이아웃 이슈 해소
- [ ] 글씨 가독성 체감 125% 향상, 회색 일변도 탈피
- [ ] "증거/대단/짜잔" 같은 오글거림 완전 제거
- [ ] 메모리/인프라 섹션이 "병목" 프레이밍으로 재구성 + 추론 시장 메모리 포함
- [ ] 하이퍼스케일러 경쟁 내러티브 추가 (치킨게임/과소투자 공포)
- [ ] 완결성·완전성·정합성 최종 검토 통과

## Non-Negotiable Principles

1. **내용 왜곡 금지** — 기존 팩트(수치·인용·출처)는 보존, 프레이밍만 재구성
2. **청중 우선** — 일반 임원이 사전지식 없이 선형으로 이해 가능해야 함
3. **점진적 확장** — 한 슬라이드는 직전 슬라이드의 개념을 확장·심화해야 함
4. **은은한 디자인** — 기존 Phosphor Icons, gradient 테마, Noto Sans KR 유지
5. **청크 I/O** — 800줄 초과 Read/Write 금지. Write는 skeleton + Edit 패턴
6. **Chrome QA 필수** — 눈으로 본 것만 완료 처리

## Scope Boundaries

### 작업 대상
- 주: `발표자료/presentation.html`
- 보조: `발표자료/tips.html` (참조 정합성 필요 시)
- 인덱스: 루트 `index.html` (필요 시 제목/리다이렉트만)

### 건드리지 않음
- 실습자료/* (Hands-on Labs)
- assets/* (이미지는 그대로 사용)
- .github/workflows/ (이미 배포 설정됨)
- 다른 브랜치/repo

### 외부 리소스
- Google Fonts, Phosphor Icons CDN — 유지
- YouTube 링크 — 유효성 검증 후 유지/교체

## Anti-Patterns (금지)

- 한 번에 800줄 이상 Write/Read
- Samsung 기업명/Galaxy 제품명을 해설 키워드로 사용
- 숫자 확대 방향으로만 변경 (inflation)
- "대단하다/짜잔/보시는 바와 같이" 류 마케팅 톤
- 개념을 중간에 되돌리기 (이미 설명한 주제를 뒤에서 재설명)
- 사실 왜곡 (수치·인용 변경 없음)

## Judgement Anchor

매 iteration 시작 시 스스로 질문:

1. "이 변경이 선형 확장 내러티브에 기여하는가?"
2. "일반 청중이 이 순서로 이해할 수 있는가?"
3. "기존 팩트를 왜곡하고 있지 않은가?"
4. "톤이 담백한가, 아니면 자기자랑처럼 보이는가?"
5. "청크 I/O 원칙을 지키고 있는가?"
